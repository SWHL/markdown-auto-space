# Marketplace Notes

## 2026-04-23

### 背景

本轮主要优化了 VS Code Marketplace 的展示与可发现性，包括：

- 重写扩展描述、分类、关键词与命令标题
- 重构 README 首屏内容与卖点结构
- 替换为更接近真实 VS Code 场景的横向宣传图
- 清理无效展示素材，收敛 VSIX 体积

### 数据对比

对比两次查看到的最近 30 天数据：

| 指标 | 调整前 | 调整后 | 变化 |
|------|------:|------:|------:|
| Acquisition | 59 | 65 | +6 |
| Till Date | 158 | 164 | +6 |
| Page views | 11 | 12 | +1 |
| Conversion Funnel | 536.36% | 541.67% | +5.31 pct |

### 当前判断

- 这轮优化已经出现正向反馈，不只是主观感觉。
- 最近 30 天 Acquisition 从 59 提升到 65，约为 `+10.2%`，说明下载端有明确增长。
- Page views 从 11 提升到 12，增幅较小，但下载增长快于浏览增长，说明这轮更像是在提升“被看到后愿意安装”的概率，以及搜索命中的质量。
- Acquisition Trend 中蓝柱（Download from Marketplace）更连续，说明下载不再只依赖零散峰值，开始出现更稳定的日常获取。
- 末段出现单日更高峰值，说明新版素材和文案至少已经带来一次明显放量。

### 风险与限制

- 当前样本仍然偏小，短期波动会放大判断偏差。
- Conversion Funnel 大于 100% 说明 Marketplace 这里的页面浏览并不覆盖全部安装路径，不能直接把它当作严格漏斗来解释。
- 因为只看到了最近 30 天的截图，还不能把这轮提升直接归因为某一个单独改动，需要继续观察。

### 后续观察点

接下来优先看 7 到 14 天：

- Page views 是否继续增长到 15 以上
- Acquisition 是否稳定在 65 以上，而不是只靠 1 到 2 天峰值抬高
- Install from VSCode 是否同步走高
- 蓝柱是否持续分布更均匀，而不是再次回到低频波动

### 结论

这轮 Marketplace 优化方向是对的。

现阶段更合理的结论不是“已经做完”，而是：

- 商店页转化和搜索承接能力有改善
- 曝光本身还没有完全打开
- 下一步应该继续做可发现性验证，而不是过早停止迭代
